데이터베이스 설계
 - 현실 세계의 업무적인 프로세서를 물리적으로 데이터베이스화 하기 위한 과정
 - 실체와 관계를 중심으로 체계적으로 표현하고 문서화하여 
      현실 세계를 컴퓨터가 처리할 수 있는 데이터로 변환

설계 순서
 - 요구조건 분석 -> 개념적 설계 -> 물리적 설계 -> 논리적 설계 -> 구현

개념적 설계
 - Entity : 업무가 관여하는 어떤 것(테이블명)
 - Relation : 업무가 관여하는 어떤 것 간의 관계
 - Attribute : 어떤 것이 가지는 성격(Entity가 가지는 속성, 컬럼)

 1. 명사를 찾아서 표시
   - 중복, 개념이 명확하지 않거나 광범위한 것, 
      특정 업무가 진행되는 과정을 나타내는 단어는 제거
      동의어는 하나로 통일
 2. 주 식별자(기본키)
   - 엔티티를 구성할 때 중요한 제약 중 하나가 그 엔티티 집합 내에서
       각 엔티티들이 서로 상이한 값의 속성을 가지고 있어야 함

※ 까치발 : ｜는 1을 의미 ○은 0을 의미 <는 여럿을 의미
    직원쪽에 까치발은 직원 입장에서 프로젝트
     프로젝트쪽에 까치발은 프로젝트 입장에서 직원

물리적 모델링
 - 물리적 설계의 목적은 개발에 사용할 데이터베이스를 선정하여 
     특정 데이터베이스로 구현될 수 있도록 구체적인 설계하는 과정
 - 엔티티 -> 테이블
 - 속성 -> 칼럼
 - 주 식별자 -> 기본키
 - 외래식별자 -> 외래키
 - 관계 -> 관계
 - 관계의 카디날리티 -> 관계의 카디날리티
 - 관계의 참여도 -> 관계의 참여도

 - 용어사전 : 논리적 데이터베이스 설계나 물리적 데이터베이스 설계시 사용되는 용어들의 의미를 정의해 놓은 문서
 	      용어는 엔티티(테이블) 이름, 속성(칼럼) 이름을 의미하기에 
                  용어사전은 즉 엔티티와 속성 이름의 의미를 설명해 놓은 사전

 - 테이블 기술서 : 개별 테이블의 자세한 문서화 수단
	             
Create table문
 - 테이블명 및 컬럼명 정의 규칙
   1. 문자(A-Z, a-z)로 시작해야 하며 30자 이내
   2. 문자(A-Z, a-z), 숫자(0-9), 특수문자(_,$,#)만 사용 가능
   3. 대소문자 구별이 없음, 소문자로 저장하려면 ' '로 묶는다.
   4. 중복 불가능

대부분의 DB는 Primary Key가 테이블당 1개
하지만 오라클에서는 경우에 따라 Primary Key를 테이블당 여러개가 가능하다.
ex) CONSTRAINT 테이블명_PK(제약조건명) PRIMARY key(컬럼명1, 컬럼명2)
외래키의 경우 여러개 만들 때
CONSTRAINT assign_FK FOREIGN key(emp_no) REFERENCES employee(emp_no),
CONSTRAINT assign_FK1 FOREIGN key(pro_no) REFERENCES project(pro_no));
제약조건명이 중복되면 안된다.

테이블의 구조보기
desc 테이블명;

ALTER TABLE문
 - 테이블 구조를 변경(추가, 변경, 삭제)
 - 추가
      alter table 테이블명 add 컬럼명 타입();
 - 변경
      alter table 테이블명 modify 컬럼명 타입();
 - 삭제
      alter table 테이블명 drop column 컬럼명;

 - SET UNUSED : 시스템의 요구가 적을 때 컬럼을 제거할 수 있도록 
	            하나 이상의 컬럼을 UNUSED로 표시
	            실제로 테이블에서 해당 컬럼이 제거되진 않음
		alter table 테이블명 set unused (컬럼명); 또는
		alter table 테이블명 set unused column 컬럼명;
                        - unused된 컬럼을 한번에 지우기
		     alter table emp01 drop unused columns;
                        unused된 컬럼을 다시 테이블에 보이게 할 순 없다.
		※ drop column 컬럼명;으로 삭제하면 Redo 와 Undo 를 발생시켜서 시간이 오래걸리기 때문에
		      unused를 사용해서 체크만 하고 한번에 삭제하는 것

테이블명 바꾸기
   rename 원래테이블명 to 바꿀테이블명;

테이블 지우기
   drop table 테이블명

테이블 모든 자료(레코드) 지우기
   truncate table 테이블명;

이 계정의 모든 테이블 보기
   select table_name from user_tables;

데이터 딕셔너리
 - 사용자가 테이블을 생성하거나 사용자를 변경하는 등의 작업을 할 때 
      데이터베이스 서버에 의해 자동으로 갱신되는 테이블
      사용자는 데이터 딕셔너리의 내용을 직접 수정하거나 삭제할 수 없고,
      사용자가 이해할 수 있는 데이터를 산출해 줄 수 있도록 하기 위해서 읽기 전용 뷰 형태로 정보를 제공
 - user_ : 자신의 계정이 소유한 객체 등에 관한 정보 조회
 - all_ : 자신 계정 소유 또는 권한을 부여 받은 객체 등에 관한 정보 조회
 - dba_ : 데이터베이스 관리자만 접근 가능한 객체 등의 정보 조회
